# 视频语料蒸馏人设 — 标准操作流程 (SOP)

本手册记录从"一个人的视频"到"可使用的 system prompt"的完整流程，适用于任何视频博主/知识类创作者的人设蒸馏。

---

## 总览

```
视频文件 → 字幕提取(V2T-Core) → 语料合并(merge_corpus.py) → 人设提炼(Claude) → 验证迭代
```

预计耗时：视频处理取决于数量和长度，人设提炼约 30 分钟。

---

## Step 1: 视频采集

### 来源
- 抖音/YouTube/B站等平台的公开视频
- 使用 `yt-dlp` 或平台专用工具下载

### 数量建议
| 目标 | 视频数量 | 字数规模 |
|------|----------|----------|
| 基本人设（语言风格） | 30-50 篇 | 2-5 万字 |
| 完整个人设（价值观+风格） | 80-150 篇 | 5-15 万字 |
| 深度蒸馏（可微调级别） | 200+ 篇 | 20 万字以上 |

### 注意事项
- 优先选择**独白/观点类**视频，避免纯剪辑/配乐类
- 覆盖不同主题（如财富、教育、职场、生活），避免单一话题偏差
- 视频时长 1-5 分钟最佳，太短信息密度低，太长噪声多

---

## Step 2: 字幕提取

### 使用 V2T-Core

```bash
# 启动 Web 服务（批量上传）
python3 web_app.py
# 浏览器打开 http://localhost:8000

# 或命令行逐个处理
python3 extract_subtitle.py -i "视频.mp4" -o "./outputs" -m small
```

### 关键配置
- **模型**：`small` 足够（速度快，中文够用）
- **线程数**：Apple M4 用 8，Intel i7 用 16-20
- **语言**：已硬编码 `language="zh"` + `initial_prompt="以下是普通话的句子。"`，确保输出简体

### 产物
- `.txt` 文件自动保存到 `outputs/txt/` 目录
- `.srt` 文件保留在 `outputs/<task_id>/` 目录

---

## Step 3: 语料合并

### 运行合并脚本

```bash
python3 distill/merge_corpus.py <txt文件目录> -o <输出路径>

# 示例：合并某人的所有文稿
python3 distill/merge_corpus.py outputs/txt/ -o distill/personas/某人/corpus.txt
```

### 脚本功能
- 自动从文件名提取标题（去掉 task_id 前缀和分辨率后缀）
- 按内容 hash 去重
- 输出统计信息（篇数、总字数）

### 前置检查
合并前抽查 3-5 篇 .txt 文件，确认：
- [ ] 是简体中文，不是繁体
- [ ] 内容可读，不是乱码或纯噪声
- [ ] 没有大量重复文件

---

## Step 4: 人设提炼

### 方法：将 corpus.txt 提交给 Claude，使用以下 prompt 模板

```markdown
请阅读以下语料（共 N 篇，约 M 万字），提炼出这个说话人的完整人设画像，输出为一份结构化的 system prompt。

要求分析以下维度：

1. **基本身份** — 年龄、职业背景、家庭状况、生活状态
2. **语言风格** — 口语化程度、句式习惯、语气词、修辞手法、是否混用外语
3. **核心价值观** — 分领域总结：财富观、教育观、家庭观、职场观、人生哲学
4. **叙事模式** — 开场方式、论证逻辑、举例习惯、收尾风格
5. **常用素材** — 反复引用的人物、案例、比喻、典故
6. **禁区** — 他不会说什么、不持什么立场、避免什么表达方式
7. **示例输出** — 2-3 段模仿此人风格的示例文本

输出格式：一份可以直接作为 system prompt 使用的 markdown 文档。
```

### 注意事项
- 如果语料超过 10 万字，分批提交（先分析 1-50 篇，再分析 51-100 篇，最后合并）
- 提炼完成后，让 Claude 用生成的 prompt 写 2-3 段话，与原文对比验证
- 关注**高频词**和**独特表达**，这些是人设的灵魂

---

## Step 5: 验证与迭代

### 验证方法

1. **风格对比测试**
   - 给出一个话题，让 Claude 用 system prompt 写一段话
   - 与该博主原视频文稿并排对比
   - 检查：句式、用词、语气、逻辑结构是否相似

2. **盲测**
   - 让不知情的人阅读生成文本和原视频文稿
   - 问：这两段话是同一个人写的吗？

3. **边界测试**
   - 问一个此人从未谈过的话题
   - 检查是否能合理延伸，而非编造与其人设矛盾的内容

### 常见问题与修复

| 问题 | 原因 | 修复方式 |
|------|------|----------|
| 生成文本太文绉绉 | 语料中书面语占比高 | 在 prompt 中强调"口语化"，增加语气词示例 |
| 价值观偏移 | 语料话题单一 | 补充更多主题的视频 |
| 说话像AI | 缺少个人细节 | 在 prompt 中增加更多个人经历和口头禅 |
| 过度使用金句 | prompt 中强调太多"金句" | 降低金句频率要求，增加过渡句示例 |

## 目录结构规范

```
distill/
├── merge_corpus.py              # 合并脚本（复用）
├── SOP.md                       # 本文件
└── personas/
    ├── xieyuan/                 # 谢员外观世界
    │   ├── corpus.txt           # 合并语料
    │   └── prompt.md            # 人设 system prompt
    ├── another_person/          # 下一个人
    │   ├── corpus.txt
    │   └── prompt.md
    └── ...
```

蒸馏新人时：
```bash
mkdir -p distill/personas/<人物名>
python3 distill/merge_corpus.py <txt目录> -o distill/personas/<人物名>/corpus.txt
# 然后按 Step 4 提炼 prompt，保存到 distill/personas/<人物名>/prompt.md
```

## 进阶：从 prompt 到微调

如果你的语料量足够大（200+ 篇，20 万字以上），可以考虑用语料微调模型：

1. 将 corpus.txt 整理为对话格式（问答对）
2. 使用 LoRA 在开源模型（如 Qwen、Llama）上微调
3. 微调后的模型天然具备此人风格，无需长 system prompt

但这需要 GPU 资源和 ML 工程能力，大多数场景下 system prompt 已经够用。
